# Tableau Setup Guide — Customer Churn &amp; RFM Dashboard

This guide rebuilds the same analysis shown in `dashboard/index.html` inside Tableau (Desktop or the free Tableau Public), using the RFM table and cohort retention table already generated by the Python pipeline.

## 1. Connect to the data

1. Open Tableau → **Connect → Text File**
2. Load `data/rfm_table.csv` (one row per customer: Recency, Frequency, Monetary, scores, Segment)
3. Add a second connection to `data/cohort_retention.csv` (for the retention heatmap) — these don't need to be joined; build them as separate sheets/data sources.

*(Tableau Public users: both CSVs work directly, no database driver needed — this is the easiest path if you don't have Tableau Desktop licensed.)*

## 2. Build the RFM segment view

1. New worksheet → drag **Segment** to Rows.
2. Drag **Monetary** to Columns, set aggregation to **SUM** → this becomes your "Revenue by Segment" bar chart.
3. Drag **CustomerID** to Label, set aggregation to **Count Distinct** → shows customer count per segment alongside revenue.
4. Sort descending by SUM(Monetary) to match the dashboard's revenue-ranked ordering.
5. Color by **Segment** for visual consistency with the HTML dashboard's palette.

## 3. Build the cohort retention heatmap

1. New worksheet → connect to `cohort_retention.csv`.
2. Drag **CohortMonth** to Rows.
3. Drag **CohortIndex** to Columns.
4. Drag **RetentionRate** to Color (and to Label, formatted as a percentage).
5. Choose a sequential color palette (e.g., Tableau's "Green" sequential) — darker = higher retention, matching the HTML dashboard's shading logic.
6. This recreates the classic cohort-retention heatmap used industry-wide for subscription and repeat-purchase businesses.

## 4. Build supporting KPI cards

Create a **Text table** or **BANs (Big Ass Numbers)** view for:
- Total customers: `COUNTD([CustomerID])` on rfm_table
- Total revenue: `SUM([Monetary])`
- Churn rate: create a calculated field —
  ```
  IF [Segment] = "Hibernating / Churned" OR [Segment] = "At Risk" OR [Segment] = "Cannot Lose Them"
  THEN 1 ELSE 0 END
  ```
  then show `AVG()` of this field × 100 as a percentage.

## 5. Assemble the dashboard

1. **Dashboard → New Dashboard**
2. Add the Segment Revenue sheet, the Cohort Heatmap sheet, and the KPI text tables onto one canvas.
3. Add a **Filter action**: clicking a segment in the bar chart filters the cohort heatmap (if you extend the analysis with a customer-ID-to-cohort join) — this is the kind of cross-filtering interactivity that's genuinely hard to replicate in a static HTML dashboard, and worth highlighting in an interview as "what Tableau adds over a lightweight web dashboard."

## 6. Publish for your portfolio

- **Tableau Public** (free): File → Save to Tableau Public → you get a shareable public link to embed in your GitHub README and resume.
- If using licensed Tableau Desktop: File → Export as PDF/Image, and save the `.twbx` packaged workbook into this `tableau/` folder so recruiters can open it directly.

## Why this matters for interviews

The same RFM logic is demonstrated three ways in this repo — Python/pandas (`notebook/02_rfm_and_cohort.py`), pure SQL (`sql/rfm_queries.sql`), and now Tableau — which lets you speak to whichever tool a JD emphasizes, and shows you understand the *underlying logic* well enough to reimplement it in any tool, not just click through one platform's wizard.
